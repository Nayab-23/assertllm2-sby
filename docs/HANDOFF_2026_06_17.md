## Phase 4 Backend Work

Implemented core Phase 4 backend parity pieces.

### Formal Types

`assertllm2_sby/formal_types.py` now includes:

- New status values:
  - `ELABORATION_ERROR`
  - `INFRASTRUCTURE_ERROR`
- New config fields:
  - `frontend`, default `yosys`
  - `prefer_bind`, default `True`
- `SourcePlan` carries:
  - `parameters`
  - `blackbox_modules`

### Source Planning

`assertllm2_sby/source_plan.py` now has helpers for:

- Preserving RTL file order from the official dataset config.
- Carrying include dirs, defines, parameters, and blackbox modules.
- Parsing top-module parameters and ports.
- Finding already defined modules.
- Inferring conservative blackbox stub ports from instances.
- Writing `generated_blackboxes.sv`.
- Emitting a source-plan artifact with:
  - `rtl_file_order_preserved`
  - generated files
  - blackboxes already defined in RTL
  - blackboxes stubbed by the backend

### Harness And SBY Artifacts

`assertllm2_sby/harness_builder.py` now supports:

- `FormalArtifacts` metadata for generated backend files.
- Bind-first artifact generation:
  - `checker.sv`
  - `<top>_bind.sv`
- Wrapper harness artifact generation as fallback.
- Blackbox stub inclusion.
- Parameter override commands via `chparam`.
- Optional frontend command selection:
  - `read_verilog -formal -sv`
  - `read_slang --formal`

### SBY Execution

`assertllm2_sby/sby_backend.py` now:

- Can run bind mode when a clock is supplied.
- Can run wrapper mode when a harness body is supplied.
- Records generated artifact paths in `result.json`.
- Records trace count and trace paths.
- Separates unsupported assertions, timeouts, infrastructure errors, and
  elaboration errors.
- Maps backend status to AssertLLM2-compatible vocabulary in details:
  - `proven`
  - `cex`
  - `undetermined`
  - `unprocessed`
  - `error`
- Detects a Yosys bind limitation where the generated checker module is parsed
  but removed as unused. That is classified as `ELABORATION_ERROR` so the
  caller can fall back to the wrapper harness.

### Result Parsing

`assertllm2_sby/result_parser.py` now distinguishes:

- infrastructure failures, such as missing `sby`
- elaboration failures, such as unresolved modules or duplicate modules
- unsupported frontend features
- timeout
- CEX
- bounded/proven pass
- unknown

### One-Design Runner Wiring

`assertllm2_sby/real_design.py` now:

- Writes `formal/golden_source_plan_artifact.json`.
- Includes blackbox stubs in parse/elaboration checks.
- Attempts bind-first per-assertion evaluation when a clock is available.
- Falls back to wrapper harness mode when bind artifact generation fails or
  Yosys removes the checker module.
- Carries parameters and blackbox modules into mutant source plans.
- Counts golden `GOLDEN_ELABORATION_ERROR` and
  `GOLDEN_INFRASTRUCTURE_ERROR`.
- Counts mutant `ELABORATION_ERROR` and `INFRASTRUCTURE_ERROR` separately.
